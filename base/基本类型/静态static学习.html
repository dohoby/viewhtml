静态static学习

例子1

先执行static { }代码块
或者执行static的变量实例化比如：static StaticTest st = new StaticTest();

package com.view.statictest;

public class StaticTest {
    int a = 110;    // 实例变量

    public static void main(String[] args) {
        staticFunction();
    }

    static {   //静态代码块
        System.out.println("1");
    }

    static StaticTest st = new StaticTest();

    {       // 实例代码块
        System.out.println("2");
        System.out.println("a="+a);
    }

    StaticTest() {    // 实例构造器
        System.out.println("3");
        System.out.println("a=" + a + ",b=" + b);
    }

    public static void staticFunction() {   // 静态方法
        System.out.println("4");
    }


    static int b = 112;     // 静态变量

}

结果:

1
2
a=110
3
a=110,b=0
4

分析: 注意类中可以直接有{} 这部分在该类中存在构造函数，而且有自身的实例化静态变量,
比如类为StaticTest，则该类中必须有实例化的变量static StaticTest st = new StaticTest(); 才会执行{},否则不执行，

例子2

package com.view.statictest.demo1;

//https://blog.csdn.net/ning1994724/article/details/87919945
public class Test extends Base{

    static{
        System.out.println("test static");
    }

    public Test(){
        System.out.println("test constructor");
    }

    public static void main(String[] args) {
        new Test();
    }
}

class Base{

    static{
        System.out.println("base static");
    }

    public Base(){
        System.out.println("base constructor");
    }
}

结果：

base static
test static
base constructor
test constructor

分析：

  • 有父类，则先执行父类的static代码块，再执行自己的static代码块
  • 调用构造函数时，先调用父类的构造函数，再调用自身的构造函数

https://blog.csdn.net/ning1994724/article/details/87919945

例子3

package com.view.statictest.demo2;

//https://blog.csdn.net/ning1994724/article/details/87919945
public class Test {
    Person person = new Person("Test");
    static{
        System.out.println("test static");
    }

    public Test() {
        System.out.println("test constructor");
    }

    public static void main(String[] args) {
        new MyClass();
    }
}

class Person{
    static{
        System.out.println("person static");
    }
    public Person(String str) {
        System.out.println("person "+str);
    }
}


class MyClass extends Test {
    Person person = new Person("MyClass");
    static{
        System.out.println("myclass static");
    }

    public MyClass() {
        System.out.println("myclass constructor");
    }
}

结果：

test static  --1
myclass static --2
person static --3
person Test --4
test constructor --5
person MyClass --6
myclass constructor --7

分析：

  • 先执行父类的static(标注1)，再执行自己的static(标注2)
  • 调用自己MyClass的构造函数之前，先调用父类Test的构造函数，调用父类Test构造函数时，因为Test类中有 new Person，所以先加载Person类，首先调用Person的static（标注3），再调用Person的构造函数(标注4)
  • Person构造函数调用完，Test调用自身构造函数(标注5)
  • 父类Test构造函数调用完，子类MyClass中有new Person，先调用Person的构造函数(标注6)， 接着子类MyClass调用自己的构造函数(标注7)
